5.3 panic && recover

go的panic机制类似于其他语言的异常机制，但是panic会引起程序的崩溃，因此panic一般用于严重的错误。对于大部分的错误和漏洞，应该使用go提供的错误机制。

当panic异常发生时，程序会中断运行，执行当前goroutine中被defer延迟的函数，随后程序输出日志信息，包括panic value和堆栈跟踪信息。

通常来说，不应该对panic做任何处理，但是有时我们可以从异常中恢复，或者说在程序崩溃前，做一些操作，如web服务器在运行时出现panic异常，应该在崩溃前把所有的连接都关闭，否则客户端会一直处于等待状态。

4.2 sha256算法

一个n位的哈希函数就是一个从任意长的消息到n位哈希值的映射，一个n位的加密哈希函数就是一个单向的、避免碰撞的n位哈希函数。这样的函数是目前在数字签名和密码保护当中极为重要的手段。当前比较流行的哈希函数主要有128位的MD4和MD5和160位的SHA-1，SHA-2族有着更多位的输出哈希值，破解难度更大，能够提高更高的安全性。

对于任意长度的消息，SHA256都会产生一个256位的哈希值，称作消息摘要。这个摘要相当于是个长度为32个字节的数组，通常有一个长度为64的十六进制字符串来表示，其中1个字节=8位，一个十六进制的字符的长度为4位。

计算"Blockchain"经过SHA256得到的哈希值：

输出：

625da44e4eaf58d61cf048d168aa6f5e492dea166d8bb54ec06c30de07db57e1

总体上，HSA256与MD4、MD5以及HSA-1等哈希函数的操作流程类似，待哈希的消息在继续哈希计算之前首先要进行以下两个步骤：

  • 对消息进行补位处理，是的最终的长度是512位的倍数
  • 然后以512位为单位对消息进行分块为\(M^{(1)}, M^{(2)}, ..., M^{(N)}\)

将消息进行逐个处理：从一个固定的初始哈希\(H^{(0)}\)开始，进行以下序列的计算： \[H^{(i)} = H^{(i-1)}+C_{M^{(i)}}(H^{(i-1)})\] 其中\(C\)是SHA256的压缩函数，\(+\)是\(\rm mod \;2^{32}\)加法，\(H^{(N)}\)是消息区块的哈希值

Map的迭代顺序是不确定的，并且不同的哈希函数实现可能导致不同的遍历顺序。在实践中，遍历的顺序是随机的，每一次遍历的顺序都不相同。这是故意的，每次都使用随机的遍历顺序可以强制要求程序不会依赖具体的哈希函数实现。如果要按顺序遍历key/value对，我们必须显式地对key进行排序，可以使用sort包的Strings函数对字符串slice进行排序。
